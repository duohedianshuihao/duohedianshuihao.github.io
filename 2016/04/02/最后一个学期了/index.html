最后一个学期了

发表于 2016-04-02 | 分类于 文字 |

已经开学一个月了，本来有几次想更新一下博客，但是都因为太懒……好多当时想写点东西已经想不起来了，就算想起来可能也没有当时那种想要表达的冲动了。不过没关系，我现在又有好多话要说。而且最后一个学期大学就结束了，我还挺想多记录一下最后的这段时光的。

出国怎么样了

悲惨的现状

没错，是悲惨，为什么呢？因为还没有学上……为什么呢？倒不是没有ad，UWindsor倒是来了EE的MAsc的录取(学院网站写的是ECE，不然我可能都不会报这个学校)。但是具体的情况还没有问清楚，比如去的话跟哪个老师，做什么，有没有人愿意给我钱，blabla……但是目前比较清楚地一点就是，应该不会去这个学校。为什么呢？首先就是我虽然没有强烈的名校情结，但是还是想去一个尽可能好的学校。别说虚荣啦，去哪还是看自己啦，我豆是虚荣豆是想去个好点的……还有就是专业，这个去了很可能是继续学EE，具体学啥我还真不知道。我片面的理解就是EE比较不容易找工作……所以还是想学个软件相关的。当初就听人说不要报保底的学校了，就算来了你也不想去。但是自己还是没忍住。可能当时的想法就是希望到时候能让我体会一下来ad的赶脚。这么想的话，那几百块钱的报名费也算是值了。

我紧张吗?

说实话还好啦。虽然现在大部分人毕业以后都有了比较明确的去向，但是我感觉自己现在不是很在意别人怎么样，毕竟别人怎么样别人才最清楚，我更想关注自己的事情。而且才四月好伐，去看看寄托的offer榜，很多授课型的都是五月份左右来通知。当然了，那几个研究型的我感觉已经凶多吉少了……说到这我就不得不说，许多事情不到自己真正做，你永远不知道自己有多懒！从寒假回家带现在，自己套磁的邮件真的是少得可怜，大概也就几封吧。说出来都对不起观众。而且没有什么实质性的回复，收到的都是套磁模板。所以说，其实自己并没有成功套磁。一方面自己水平有限，套磁没什么可说的(其实是一个借口，没什么可说的你ps cv 怎么写的？sb)。另一方面，心里还有一个声音-“我不适合做科研，我就是想找工作，所以授课型的更适合我！”。不知道这算不算给自己安排简单的路来走，这样叫不叫逃避。我倒是希望自己不要后悔就好。嗯。

有什么打算

其实那两个授课型的ECE还是比较有希望的。我是说UCalgary和UWO。尤其是UWO，给人的感觉就是大公共汽车==#(话糙理不糙……)而且UWO的ECE是有SE的方向的，就是不知道是不是还要选一些EE的课程。UWO的话离多伦多也不是很远，工作问题应该也容易些，好吧我猜的。

但是最近又有了一个新的打算，如果这次放弃，重新申请春季入学怎么样？这是一个比较大胆的想法，我也没有完全想好。这样的话，我可以有机会尝试一下Waterloo的ECE和SYDE的MEng，感觉这两个专业还是比较友好的。当初因为感觉水卢会很难申，但现在感觉其实没有想象中的那么困难，据说它在扩招，主要是不想因为这样就错过了。而且…Waterloo说出去给人一种NB的赶脚……也可以尝试一下SFU的big data。申SFU CS的MSc真的是有点没必要。当初就是因为纠结签证时间的问题，放弃了很多学校的MEng。(其实我现在也不是很懂哎，有好心人看到能不能留言给我讲讲啊T—T 四学期的项目是拿几年工签？工签期间我是只要找到工作就可以吗？还是说工钱期间我要拿到绿卡才可以啊？)

当然还有一种情况就是，这次申请全部失败，俗称全聚德。那么我就可以理所应当的去准备春季申请了(尴尬).现在是打算4月中旬，中期检查以后，改改材料，把Waterloo的春季先申请了。就酱。

爱不释手的X220

还是说点让人高兴的吧，不管申请的结果怎么样，是不是又要再等半年才能走，生活都还要继续。^_^

先说pixel

说真的，上学期的时候一峰买了一个pixel，当时我还拿着把玩了一段时间，感觉不错，但是并没有那么强烈的那种我要有一个pixel的欲望。当然，那时候已经觉得pixel外观德味，屏幕毒毒毒！寒假在家逛D版，发现chromebook吧的微信群里准备搞pixel的团购。于是大批hiper火速赶往该群，其中就有我(地板小分队如数赶到！)。那时候还没有过年，团购初步定在2月24号。那时候，我就数着日子，盼星星盼月亮盼那一天。23晚上……没消息 24白天……说是明天有人去看货 25上午……终于有消息了，中午开始，我就一直守在电脑前面，等链接，等着给人家送钱！最后发现是期货，五台现货，让后60多台还是80多台期货，1400，很新。突然一个通知，让大家填信息，现在先排队，后面pixel到了再付款。马上点开链接，填信息，然后备注那里犹豫了一哈儿。马上点了确认。结果排了No.7 兴奋！第二批大概要到3.8号以后了，让后就又开始盼日子。到了10号，说是这批货都被海关没收了……团购到此结束……

两台电脑更新博客

x220的初衷就是为了能让我比较容易的把电脑带出去，不用每次都拔拔拔，回来再插插插。所以如果我想在外面写博客怎么办？因为我不是很理解github的用法，所以弄了半天才勉强搞定这个问题。所以写下来，省的下次忘了==#

  1. 首先，在github账号新建一个r远程仓库。并且在win的blog文件夹建立本地仓库，git bash运行

    git init

    然后运行

    git remote add origin git@github.com:username/xxx.git

    git add .

    git commit -m “xxx”

    git push -u origin master

    这样就把你的所有文件都放到github了。

  2. 蓝后，在x220的blog文件夹初始化一个git本地仓库，再把github里的文件全部fetch下来。

    git init

    git remote add origin git@github.com:username/xxx.git

    git fetch –all

    git reset –hard origin/master

    这样就把博客的文件从原来的电脑转到了x220.但是生成的时候会发现失败，而且hexo server用不了(其实我也记不清了…… 我记得好像是会失败==#)。然后我就

    hexo init

    npm install

    然后就发现主题啥的全变成默认的了，但是内容还在，hexo server能用了。于是我又把之前的hexo配置文件粘贴过来，以及替换主题，然后就发现。好了……

    可能有些地方是我弄错了……anyway，好了就行(憋说话，吻我)

  3. 以后再不同电脑用的时候，都要先

    git fetch –all

    git reset –hard origin/master

    然后再写博客

    hexo new “xxx”

    hexo g -d

    写完之后还要

    git add .

    git commit -m “xxx”

    git push -u origin master

    这样就能保证github上永远是最新的，每次都让github上最新的替换掉本地的，就不会丢掉文章了。
